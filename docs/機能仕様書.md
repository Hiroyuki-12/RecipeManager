# レシピ管理アプリ 機能仕様書

## 1. はじめに
本書は、レシピ管理アプリの各機能について、画面・入出力・処理内容・API 仕様を詳細に定義するものである。
要件定義書と合わせて参照すること。

---

## 2. 機能一覧

| 機能 ID | 機能名 | 概要 |
|---------|--------|------|
| F-01 | レシピ一覧表示 | 登録済みレシピを一覧表示し、検索・絞り込み・並べ替えを行う |
| F-02 | レシピ詳細表示 | 1 件のレシピの詳細を表示する |
| F-03 | レシピ新規登録 | 新規レシピを登録する |
| F-04 | レシピ編集 | 既存レシピを編集する |
| F-05 | レシピ削除 | レシピを削除する |

---

## 3. 機能詳細

### F-01 レシピ一覧表示

#### 概要
登録済みレシピの一覧を表示する。検索・絞り込み・並べ替え機能を提供する。

#### 画面項目
| 項目 | 種別 | 説明 |
|------|------|------|
| 検索ボックス | テキスト入力 | レシピ名で部分一致検索 |
| カテゴリ絞り込み | プルダウン | 「全て」「和食」「洋食」「中華」「その他」 |
| 並べ替え | プルダウン | 「新しい順」「名前順」 |
| 新規登録ボタン | ボタン | 登録画面へ遷移 |
| レシピリスト | 一覧 | レシピ名、カテゴリ、調理時間を表示 |

#### 動作
- 画面表示時に `GET /api/recipes` を呼び出し、一覧を取得する
- 検索ボックスへの入力、絞り込み・並べ替えの変更時に再取得する
- レシピをクリックすると詳細画面へ遷移する(`/recipes/:id`)
- 「新規登録」ボタンクリックで登録画面へ遷移する(`/recipes/new`)

#### 入力チェック
- 検索キーワードは最大 100 文字

#### エラーハンドリング
- API エラー時は画面上にエラーメッセージを表示

---

### F-02 レシピ詳細表示

#### 概要
選択したレシピの全項目を表示する。

#### 画面項目
| 項目 | 種別 | 説明 |
|------|------|------|
| レシピ名 | 表示 | |
| カテゴリ | 表示 | |
| 調理時間 | 表示 | 分単位で表示 |
| 何人分 | 表示 | |
| 材料 | リスト | 材料名と「数量+単位」を一覧表示(例:`玉ねぎ … 1個`、`塩 … 少々`) |
| 手順 | リスト | 番号付きで表示 |
| メモ | 表示 | 改行を保持して表示 |
| 編集ボタン | ボタン | 編集画面へ遷移 |
| 削除ボタン | ボタン | 削除確認ダイアログを表示 |
| 戻るボタン | ボタン | 一覧画面へ戻る |

#### 動作
- 画面表示時に `GET /api/recipes/:id` を呼び出す
- 編集ボタンで `/recipes/:id/edit` へ遷移
- 削除ボタンで確認ダイアログ → OK で削除 API 実行 → 一覧へ遷移

#### エラーハンドリング
- 該当 ID のレシピが存在しない場合、「レシピが見つかりません」メッセージを表示

---

### F-03 レシピ新規登録

#### 概要
新規レシピを登録する。

#### 画面項目
| 項目 | 種別 | 必須 | 説明 |
|------|------|------|------|
| レシピ名 | テキスト | ○ | 最大 100 文字 |
| カテゴリ | プルダウン | - | 和食 / 洋食 / 中華 / その他 |
| 調理時間(分) | 候補付き数値入力 | - | 0 以上の整数。候補:5 / 10 / 15 / 20 / 30 / 45 / 60 / 90 / 120。候補にない値も自由入力可 |
| 何人分 | 候補付き数値入力 | - | 1 以上の整数。候補:1〜10。候補にない値も自由入力可 |
| 材料(動的) | 入力行 | ○ | 「材料名 / 数量 / 単位」の 3 要素を 1 行とし、行の追加・削除が可能(最低 1 行)。単位は候補(個 / g / kg / ml / cc / L / 本 / 枚 / 杯 / 房 / 片 / 大さじ / 小さじ / カップ / 少々 / 適量)から選択でき、自由入力も可 |
| 手順(動的) | 入力行 | ○ | 「手順内容」を 1 行とし、行の追加・削除が可能(最低 1 行)。番号は表示時に自動採番 |
| メモ | テキストエリア | - | |
| 登録ボタン | ボタン | - | 入力内容を送信 |
| キャンセルボタン | ボタン | - | 一覧画面へ戻る |

#### 動作
- 「+ 材料を追加」「+ 手順を追加」ボタンで入力行を増やせる
- 各行の「削除」ボタンで行を削除できる
- 登録ボタン押下で入力チェック → `POST /api/recipes` を呼び出す
- 登録成功時は一覧画面へ遷移する

#### 入力チェック
- レシピ名:必須、最大 100 文字
- 材料:1 件以上必須。「材料名」が入力されている行のみを有効行とする(数量・単位は任意)
- 手順:1 件以上必須、各行の内容は必須
- 調理時間:任意、0 以上の整数
- 何人分:任意、1 以上の整数
- 数量:任意。単位が「少々」「適量」など数量を伴わないものの場合、空欄でも登録可

#### エラーハンドリング
- バリデーションエラー時は該当項目の下にエラーメッセージを表示
- API エラー時は画面上にエラーメッセージを表示

### F-04 レシピ編集

#### 概要
既存レシピの内容を編集する。

#### 画面項目
新規登録画面と同じ。初期値として既存レシピの内容を表示する。

#### 動作
- 画面表示時に `GET /api/recipes/:id` で初期値を取得しフォームに反映
- 更新ボタン押下で `PUT /api/recipes/:id` を呼び出す
- 更新成功時は詳細画面へ遷移

#### 入力チェック
新規登録画面と同じ。

---

### F-05 レシピ削除

#### 概要
選択したレシピを削除する。

#### 動作
- 詳細画面の「削除」ボタン押下で確認ダイアログを表示
  - 「本当に削除しますか?」
- OK 押下で `DELETE /api/recipes/:id` を呼び出す
- 削除成功時は一覧画面へ遷移

#### エラーハンドリング
- API エラー時はエラーメッセージを表示

## 4. API 仕様詳細

### 4.1 共通仕様
- ベース URL:`/api`
- リクエスト・レスポンスは JSON 形式
- 文字コード:UTF-8

### 4.2 GET /api/recipes(一覧取得)

#### クエリパラメータ
| 名前 | 型 | 必須 | 説明 |
|------|----|------|------|
| keyword | string | - | レシピ名の部分一致検索 |
| category | string | - | カテゴリで絞り込み |
| sort | string | - | `created_desc`(デフォルト) / `name_asc` |

#### レスポンス例
```json
[
  {
    "id": 1,
    "title": "肉じゃが",
    "category": "和食",
    "cooking_time": 30,
    "created_at": "2026-05-01T10:00:00Z"
  },
  {
    "id": 2,
    "title": "カレー",
    "category": "洋食",
    "cooking_time": 45,
    "created_at": "2026-05-02T12:00:00Z"
  }
]
```

### 4.3 GET /api/recipes/:id(詳細取得)

#### レスポンス例
```json
{
  "id": 1,
  "title": "肉じゃが",
  "category": "和食",
  "ingredients": [
    { "name": "玉ねぎ", "amount": "1", "unit": "個" },
    { "name": "じゃがいも", "amount": "3", "unit": "個" },
    { "name": "塩", "amount": "", "unit": "少々" }
  ],
  "steps": [
    { "order": 1, "description": "材料を切る" },
    { "order": 2, "description": "炒める" }
  ],
  "servings": 4,
  "cooking_time": 30,
  "memo": "甘めが好み",
  "created_at": "2026-05-01T10:00:00Z",
  "updated_at": "2026-05-01T10:00:00Z"
}
```

#### エラー
- 404:該当 ID のレシピが存在しない

### 4.4 POST /api/recipes(新規作成)

#### リクエスト例
```json
{
  "title": "肉じゃが",
  "category": "和食",
  "ingredients": [
    { "name": "玉ねぎ", "amount": "1", "unit": "個" }
  ],
  "steps": [
    { "order": 1, "description": "材料を切る" }
  ],
  "servings": 4,
  "cooking_time": 30,
  "memo": "甘めが好み"
}
```

#### レスポンス
- 201 Created:作成されたレシピを返す(詳細取得と同じ形式)
- 400 Bad Request:バリデーションエラー

### 4.5 PUT /api/recipes/:id(更新)

#### リクエスト
POST と同じ形式

#### レスポンス
- 200 OK:更新後のレシピを返す
- 400 Bad Request:バリデーションエラー
- 404 Not Found:該当 ID のレシピが存在しない

### 4.6 DELETE /api/recipes/:id(削除)

#### レスポンス
- 204 No Content:削除成功
- 404 Not Found:該当 ID のレシピが存在しない

---

## 5. バリデーション仕様

| 項目 | ルール |
|------|--------|
| title | 必須、最大 100 文字 |
| category | 任意、最大 50 文字 |
| ingredients | 必須、1 件以上、各要素の `name` は必須。`amount` / `unit` は任意(単位が「少々」「適量」等の場合 `amount` は空可) |
| steps | 必須、1 件以上、各要素の `description` は必須 |
| servings | 任意、1 以上の整数 |
| cooking_time | 任意、0 以上の整数 |
| memo | 任意 |
