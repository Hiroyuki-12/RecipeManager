# レシピ管理アプリ 画面設計書

## 1. はじめに
本書は、レシピ管理アプリの各画面の UI レイアウト・項目・操作を定義するものである。

---

## 2. 画面一覧

| 画面 ID | 画面名 | URL | 説明 |
|---------|--------|------|------|
| S-01 | レシピ一覧画面 | `/` または `/recipes` | レシピ一覧、検索、絞り込み、並べ替え |
| S-02 | レシピ詳細画面 | `/recipes/:id` | レシピの詳細表示 |
| S-03 | レシピ登録画面 | `/recipes/new` | 新規レシピの登録 |
| S-04 | レシピ編集画面 | `/recipes/:id/edit` | 既存レシピの編集 |

---

## 3. 共通レイアウト

```
┌──────────────────────────────────────────┐
│  🍳 レシピ管理アプリ          [+ 新規登録] │  ← ヘッダー(全画面共通)
├──────────────────────────────────────────┤
│  ← 戻る (画面に応じて表示 / Breadcrumb)  │
│                                          │
│           (ページごとのコンテンツ)       │
│                                          │
└──────────────────────────────────────────┘
```

- ヘッダーのタイトルをクリックすると一覧画面へ戻る
- ヘッダー右側に「+ 新規登録」ボタンを常設する（登録画面 `/recipes/new` 表示時のみ非表示）
- 戻り導線は `components/Breadcrumb.tsx` で共通化。詳細画面では「← 一覧へ戻る」、編集画面では「← 詳細へ戻る」と文脈に応じて変動する

## 4. 画面詳細

### S-01 レシピ一覧画面

#### レイアウト
```
┌────────────────────────────────────────────────────────┐
│  🍳 レシピ管理アプリ                       [+ 新規登録] │
├────────────────────────────────────────────────────────┤
│                                                        │
│  [🔍 レシピ名で検索 ____________________]              │
│  カテゴリ:[全て ▼]   並び順:[新しい順 ▼]               │
│  ──────────────────────────────────────────────        │
│  ┌──────────────┐  ┌──────────────┐  ┌──────────────┐  │
│  │ 肉じゃが      │  │ カレー        │  │ 麻婆豆腐      │  │
│  │ 和食          │  │ 洋食          │  │ 中華          │  │
│  │ 🍴4人 ⏱30分  │  │ 🍴4人 ⏱45分  │  │ 🍴3人 ⏱20分  │  │
│  │ 🥕材料 5     │  │ 🥕材料 7     │  │ 🥕材料 4     │  │
│  │ 更新 5/01    │  │ 更新 5/02    │  │ 更新 5/03    │  │
│  └──────────────┘  └──────────────┘  └──────────────┘  │
│                                                        │
└────────────────────────────────────────────────────────┘
```

カードレイアウトは CSS Grid `repeat(auto-fill, minmax(280px, 1fr))` でレスポンシブに 1〜N 列に並ぶ（[frontend/styles/globals.css](../frontend/styles/globals.css)）。

#### 画面項目
| No | 項目 | 種別 | 動作 |
|----|------|------|------|
| 1 | 検索ボックス | テキスト入力 | 入力で部分一致検索を実行（**300ms デバウンス**で API 呼び出し） |
| 2 | カテゴリプルダウン | プルダウン | 「全て / 和食 / 洋食 / 中華 / その他」 |
| 3 | 並び順プルダウン | プルダウン | 「新しい順 / 名前順」 |
| 4 | レシピカード | カード | クリックで詳細画面へ遷移。表示項目: タイトル / カテゴリ / 🍴何人分 / ⏱調理時間 / 🥕材料数 / 更新日 |

※ 「+ 新規登録」ボタンは画面ツールバーから廃止し、共通ヘッダー右側に常設（[frontend/components/Header.tsx](../frontend/components/Header.tsx)）。

関連実装: [frontend/pages/recipes/index.tsx](../frontend/pages/recipes/index.tsx)

#### 表示時の状態
- 初期表示:登録順(新しい順)で全件表示
- データなし時:「レシピがありません」と表示
- 検索結果なし時:「該当するレシピがありません」と表示

### S-02 レシピ詳細画面

#### レイアウト
```
┌──────────────────────────────────────────────────┐
│  🍳 レシピ管理アプリ                [+ 新規登録]  │
├──────────────────────────────────────────────────┤
│                                                  │
│  ← 一覧へ戻る  (Breadcrumb)                       │
│                                                  │
│  肉じゃが                       [編集] [削除]    │
│  カテゴリ:和食                                   │
│  調理時間:30分    何人分:4人分                  │
│  ────────────────────────────────────────        │
│  ## 材料                                         │
│   ・玉ねぎ ・・・・・ 1個                        │
│   ・じゃがいも ・・・ 3個                        │
│   ・牛肉 ・・・・・・ 200g                       │
│   ・塩 ・・・・・・・ 少々                       │
│                                                  │
│  ## 作り方                                       │
│   1. 材料を一口大に切る                          │
│   2. 鍋で炒める                                  │
│   3. 調味料を加えて煮込む                        │
│                                                  │
│  ## メモ                                         │
│   甘めが好み。砂糖を多めに。                     │
│                                                  │
└──────────────────────────────────────────────────┘
```

セクション見出しは `<h3 class="section-title">` で「材料」「作り方」「メモ」を表示する（プロトタイプの `【材料】【手順】【メモ】` 表記から変更）。

#### 画面項目
| No | 項目 | 種別 | 動作 |
|----|------|------|------|
| 1 | 戻るリンク（Breadcrumb） | リンク | 一覧画面へ遷移 |
| 2 | 編集ボタン | ボタン | 編集画面へ遷移 |
| 3 | 削除ボタン | ボタン | 確認モーダル→削除→一覧画面へ |

#### ダイアログ（削除確認モーダル）
- 文言:「このレシピを削除しますか?」
- ボタン: 左に `[キャンセル]`（`btn-strong`）、右に `[削除する]`（`btn-danger-solid` で赤強調）
- 削除実行中は「削除中...」に切り替え、二重押下を防ぐ

### S-03 レシピ登録画面

#### レイアウト
```
┌──────────────────────────────────────────────────┐
│  🍳 レシピ管理アプリ                             │
├──────────────────────────────────────────────────┤
│                                                  │
│  新規レシピ登録                                  │
│  ────────────────────────────────────────        │
│                                                  │
│  レシピ名 *                                      │
│  [_______________________________]               │
│                                                  │
│  カテゴリ          調理時間(分)    何人分        │
│  [和食 ▼]          [ 30   ▼]      [ 4   ▼]      │
│  ※ 候補から選択 / 候補にない値は自由入力可       │
│                                                  │
│  材料 *                                          │
│  [玉ねぎ_______] [ 1   ] [個   ▼]  [削除]        │
│  [豚肉_________] [200  ] [g    ▼]  [削除]        │
│  [塩___________] [     ] [少々 ▼]  [削除]        │
│  ※ 単位は候補から選択 / 自由入力可                │
│  [+ 材料を追加]                                  │
│                                                  │
│  手順 *                                          │
│  1. [玉ねぎを薄切りにする_____________] [削除]   │
│  2. [フライパンで炒める_______________] [削除]   │
│  [+ 手順を追加]                                  │
│                                                  │
│  メモ                                            │
│  ┌────────────────────────────────────┐          │
│  │                                    │          │
│  │                                    │          │
│  └────────────────────────────────────┘          │
│                                                  │
│              [キャンセル]  [登録]                │
│                                                  │
└──────────────────────────────────────────────────┘
```

#### 画面項目
| No | 項目 | 種別 | 必須 | 説明 |
|----|------|------|------|------|
| 1 | レシピ名 | テキスト入力 | ○ | 最大 100 文字 |
| 2 | カテゴリ | プルダウン | - | 和食 / 洋食 / 中華 / その他 |
| 3 | 調理時間 | 候補付き数値入力 | - | 分単位、0 以上。候補:5 / 10 / 15 / 20 / 30 / 45 / 60 / 90 / 120。候補にない値も自由入力可 |
| 4 | 何人分 | 候補付き数値入力 | - | 1 以上。候補:1〜10。候補にない値も自由入力可 |
| 5 | 材料行 | テキスト入力 + 数値入力 + 候補付きテキスト入力 | ○ | 材料名 / 数量 / 単位の 3 要素、最低 1 行。単位候補:個 / g / kg / ml / cc / L / 本 / 枚 / 杯 / 房 / 片 / 大さじ / 小さじ / カップ / 少々 / 適量。候補にない単位も自由入力可。「少々」「適量」のように数量を伴わない単位の場合、数量は空欄でも可 |
| 6 | 材料追加ボタン | ボタン | - | 材料行を追加 |
| 7 | 材料削除ボタン | ボタン | - | 該当行を削除 |
| 8 | 手順行 | テキスト入力 | ○ | 最低 1 行、番号は表示時に自動採番 |
| 9 | 手順追加ボタン | ボタン | - | 手順行を追加 |
| 10 | 手順削除ボタン | ボタン | - | 該当行を削除 |
| 11 | メモ | テキストエリア | - | 複数行入力可 |
| 12 | 登録ボタン | ボタン | - | 入力チェック→登録 |
| 13 | キャンセルボタン | ボタン | - | 一覧画面へ戻る |

#### バリデーションエラー表示
- エラー対象 input に `.has-error` クラスを付与し赤枠で強調
- 該当項目の下に赤字でエラーメッセージを表示
  - 「レシピ名を入力してください」
  - 「材料を 1 件以上入力してください」
  - 「手順を 1 件以上入力してください」
- onChange でその項目のエラーをリアルタイムに解除
- 登録/更新ボタン押下時、最初のエラーフィールドへ `scrollIntoView` で自動スクロール

#### 動的行追加の追従型 UX
- 「+ 材料を追加」「+ 手順を追加」ボタンに加え、以下も対応:
  - 最終行で **Enter キー押下** → 次の空行を自動追加
  - 最終行に値が入力された時点で空行 1 行を自動追加（「+ 追加」ボタンを押さずに連続入力可）

#### 登録後の遷移
- 登録成功時は **詳細画面 `/recipes/:id`** へ遷移（登録内容をそのまま確認できるようにするため）
- キャンセルボタンは一覧画面へ戻る

関連実装: [frontend/pages/recipes/new.tsx](../frontend/pages/recipes/new.tsx) / [frontend/components/RecipeForm.tsx](../frontend/components/RecipeForm.tsx)

### S-04 レシピ編集画面

#### レイアウト
S-03 とほぼ同じ。タイトルが「レシピ編集」になり、ボタンが「更新」となる。
初期値として既存レシピの内容が表示される。

```
┌──────────────────────────────────────────────────┐
│  🍳 レシピ管理アプリ                             │
├──────────────────────────────────────────────────┤
│                                                  │
│  レシピ編集                                      │
│  ────────────────────────────────────────        │
│                                                  │
│  (S-03 と同じ入力フォーム、初期値あり)          │
│                                                  │
│              [キャンセル]  [更新]                │
│                                                  │
└──────────────────────────────────────────────────┘
```

#### 動作の差分
- 画面表示時に既存レシピを取得してフォームへセットする
- 更新成功時は詳細画面へ遷移
- バリデーション挙動・動的行追加・エラー表示は S-03 と同じ

関連実装: [frontend/pages/recipes/[id]/edit.tsx](../frontend/pages/recipes/[id]/edit.tsx) / [frontend/components/RecipeForm.tsx](../frontend/components/RecipeForm.tsx)

```
        ┌────────────────┐
        │  一覧画面       │ ◀──────┐
        │  (S-01)        │         │

          │            │           │

   └──────┬───┘  └─────┬────┘      │
          │            │           │
```

## 6. デザイン方針
- シンプル・ミニマルな UI
- レスポンシブ対応は本リリースでは対象外(PC ブラウザ前提)
- 配色は白基調、アクセントカラーは未定(実装フェーズで決定)
