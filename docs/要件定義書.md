# レシピ管理アプリ 要件定義書

## 1. プロジェクト概要

### 1.1 プロジェクト名
レシピ管理アプリ(RecipeManager)

### 1.2 目的
個人利用を前提とした、シンプルなレシピ管理 Web アプリケーションを開発する。
日々の料理レシピを Web 上で登録・閲覧・編集・削除できるようにし、
材料や手順を構造的に保存することで、将来的な機能拡張(買い物リスト連携等)にも備える。

### 1.3 背景
TaskManagement アプリで使用した技術スタック(React + Spring Boot + PostgreSQL)とは
異なる技術スタックを採用することで、新しい技術への習熟を目的とする。

### 1.4 対象ユーザー
- 本人(個人利用)
- 認証機能は設けず、単一ユーザーが利用する想定

---

## 2. スコープ

### 2.1 本リリースで実現する機能
- レシピの新規登録
- レシピの一覧表示
- レシピの詳細表示
- レシピの編集
- レシピの削除
- レシピ名による検索(部分一致)
- カテゴリによる絞り込み
- 並べ替え(新しい順 / 名前順)

### 2.2 本リリースで実現しない機能(対象外)
- ユーザー認証 / ログイン機能
- 画像アップロード
- お気に入り機能
- タグ機能
- 買い物リスト連携
- 献立カレンダー
- 栄養計算
- レシピの公開・共有(SNS 機能)

---

## 3. 技術構成

### 3.1 フロントエンド
- **Next.js**(Pages Router)
- **TypeScript**
- 全画面クライアントコンポーネント中心で構築

### 3.2 バックエンド
- **Ruby on Rails**(API mode)

### 3.3 データベース
- **MySQL**

### 3.4 インフラ
- **AWS**(TaskManagement と同等の構成を踏襲)

### 3.5 アーキテクチャ
```
[ブラウザ(Next.js)] ⇄ [Rails API] ⇄ [MySQL]
```
フロントエンドは Rails API を `fetch` で呼び出し、
Rails が MySQL へのデータアクセスと加工を担当する。

## 4. 機能要件

### 4.1 画面一覧
| No | 画面名 | 概要 |
|----|--------|------|
| 1 | レシピ一覧画面 | 登録済みレシピの一覧を表示。検索・絞り込み・並べ替えが可能 |
| 2 | レシピ詳細画面 | 1 件のレシピの詳細を表示 |
| 3 | レシピ登録画面 | 新規レシピを入力して登録 |
| 4 | レシピ編集画面 | 既存レシピの内容を編集 |

### 4.2 機能詳細

#### 4.2.1 レシピ一覧画面
- 登録済みレシピを一覧表示する
- 表示項目:レシピ名、カテゴリ、調理時間
- レシピ名で部分一致検索ができる
- カテゴリで絞り込みができる(プルダウン)
- 並べ替え:新しい順 / 名前順
- 各レシピをクリックすると詳細画面へ遷移する
- 「新規登録」ボタンから登録画面へ遷移する

#### 4.2.2 レシピ詳細画面
- 選択したレシピの全項目を表示する
- 「編集」ボタンから編集画面へ遷移する
- 「削除」ボタンでレシピを削除する(確認ダイアログあり)

#### 4.2.3 レシピ登録画面
- 各項目を入力して新規レシピを登録する
- 材料・手順は動的に行を追加・削除できる
- 必須項目が未入力の場合はエラー表示
- 登録成功後は一覧画面へ遷移する

#### 4.2.4 レシピ編集画面
- 既存レシピの内容を編集する
- 入力フォームは登録画面と同じ構成
- 更新成功後は詳細画面または一覧画面へ遷移する

## 5. データ要件

### 5.1 テーブル構成
本アプリでは `recipes` テーブル 1 つで管理する。

### 5.2 recipes テーブル定義
| カラム名 | 型 | NULL | 説明 |
|----------|----|----|------|
| id | BIGINT | NOT NULL | 主キー(自動採番) |
| title | VARCHAR(100) | NOT NULL | レシピ名 |
| category | VARCHAR(50) | NULL | カテゴリ(和食 / 洋食 / 中華 等) |
| ingredients | JSON | NOT NULL | 材料(JSON 配列) |
| steps | JSON | NOT NULL | 手順(JSON 配列) |
| servings | INT | NULL | 何人分 |
| cooking_time | INT | NULL | 調理時間(分) |
| memo | TEXT | NULL | メモ |
| created_at | DATETIME | NOT NULL | 作成日時 |
| updated_at | DATETIME | NOT NULL | 更新日時 |

### 5.3 JSON 構造

#### ingredients(材料)
```json
[
  { "name": "玉ねぎ", "amount": "1個" },
  { "name": "豚肉",   "amount": "200g" },
  { "name": "醤油",   "amount": "大さじ2" }
]
```

#### steps(手順)
```json
[
  { "order": 1, "description": "玉ねぎを薄切りにする" },
  { "order": 2, "description": "フライパンで炒める" },
  { "order": 3, "description": "調味料を加えて煮込む" }
]
```

## 6. API 仕様(概要)

| メソッド | パス | 用途 |
|----------|------|------|
| GET | /api/recipes | レシピ一覧取得(検索・絞り込み・並べ替えクエリ対応) |
| GET | /api/recipes/:id | レシピ詳細取得 |
| POST | /api/recipes | レシピ新規作成 |
| PUT | /api/recipes/:id | レシピ更新 |
| DELETE | /api/recipes/:id | レシピ削除 |

### クエリパラメータ(GET /api/recipes)
| 名前 | 型 | 用途 |
|------|----|------|
| keyword | string | レシピ名の部分一致検索 |
| category | string | カテゴリでの絞り込み |
| sort | string | 並び順(`created_desc` / `name_asc`) |

## 7. 非機能要件

### 7.1 性能
- 個人利用前提のため、想定レコード数は数百〜数千件程度
- 一覧表示は 1 秒以内に応答

### 7.2 セキュリティ
- 個人利用かつ AWS 上で限定公開のため、認証は設けない
- ただし、API は SQL インジェクション・XSS 対策を Rails の標準機能で行う

### 7.3 ブラウザ対応
- Google Chrome(最新版)
